# cEmoji V0.0.10

## 更新内容
- 新增右键置顶表情包功能，置顶状态会保存并在下次启动后继续生效。
- 置顶表情包会显示左上角“置顶”角标，GIF 表情继续显示右上角 `GIF` 角标，两个角标互不冲突。
- 新增 GIF 悬停播放：鼠标停留在 GIF 表情上一秒后播放缩略图动画，移开后自动停止。
- 新增导入结果统计，导入完成后提示成功导入、已存在跳过和失败数量。
- 图片、ZIP 和剪贴板导入新增内容去重，同图不同名不会重复导入。
- 新增“清空所有表情包”按钮，可一次清空原图、缩略图和置顶记录。
- 发布包结构调整为外层启动器加 `app` 运行目录，减少用户打开目录时看到的大量依赖文件。

## 体验优化
- 图片显示改为手绘缩略图，大图等比缩小，小图保持原始尺寸居中显示，不再被强行放大。
- 多图片和多 ZIP 导入时显示固定宽度进度窗口，长文件名自动省略，窗口不会被撑宽。
- 导入进度窗口会相对主窗口居中显示。
- 底部操作提示更新为“鼠标左键单击图片可复制到剪切板，右键可置顶”。
- 深色主题下优化表情文件名和鼠标悬停背景色，避免黑字或过亮 hover 背景。
- 启动链路延迟导入 Pillow 和上传模块，减少冷启动耗时。
 
## BUG修复
- 修复图片和 ZIP 内日文文件名导入后界面显示乱码的问题。
- 修复手绘缩略图后列表项点击区域不正确，导致图片无法点击复制的问题。
- 修复清空表情包提示数量把原图和缩略图重复计数的问题。
- 修复 GIF 复制支持被误移除的问题，继续保留 `cpgif.exe` 复制 GIF。
- 修复 Nuitka standalone 发布包移动目录时路径不正确的问题。
- 修复 GitHub Actions 中 Nuitka standalone 构建因 Dependency Walker 非交互下载确认失败而中断的问题。

## 清理
- 删除未使用的 `resource/icon/gif_icon.png` 和空的 `resource/icon` 目录。
