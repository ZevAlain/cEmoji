## ❌ 你不可以做什么

### 闭源使用
✗ 不能在修改后的版本中隐藏源代码
✗ 必须以相同许可证开源衍生作品

### 虚假宣传
✗ 不能声称修改的版本是官方 cEmoji
✗ 不能声称官方开发者与你的版本有关联

### 责任转移
✗ 不能声称官方承担你修改版本导致的问题

## 📋 重要条款

当你修改和分发 cEmoji 时，必须：

### 1. 标注来源
- 标明修改的源自 cEmoji
- 提供指向原项目的链接
- 保留原始许可证和著作权信息

### 2. 说明修改
- 清楚地列出主要修改内容
- 使用清晰的版本号（建议使用语义化版本）

### 3. 使用相同许可证
- 修改后的版本必须使用 GPLv3
- 不能改为其他许可证
- 依赖项必须符合兼容性要求

### 4. 保留版权信息
- 保留原始著作权声明
- 修改者对修改部分的代码享有著作权

## 🤝 贡献指南

### 提交代码前

1. 确保你同意将代码贡献给 cEmoji 项目
2. 你的代码也将遵循 GPLv3 许可证
3. 保留你对修改部分的著作权

### 提交过程

1. Fork 项目
2. 创建功能分支
3. 提交高质量的 Pull Request
4. 在 PR 中说明修改内容
5. 等待审核和合并

所有贡献者的著作权都将被保留。
